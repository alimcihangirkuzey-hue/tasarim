# ADR-004 — Katalog kanonikliği; BrandKit styva
**Durum:** KABUL · **Tadil:** D-9

Orijinal karar: katalog=swiss kanonik, BrandKit styva. D-9 tadiliyle: restoran operasyon verisi (menü dahil) SWISS'te (Repo A) yaşar; bağ = contract (kontrolsüz veri kopyası yasak; canonical owner + read-model/snapshot/event). C'deki katalog, C'nin atölye ürünü için yaşamaya devam eder; platform katalog bağları sözleşmeyle kurulur.

*Kaynak: Faz 0 §17 (tam metin styva /docs'ta); C-repo etkisi: `tasarim` kataloğu atölye-yerel kalır (M1/M5 anayasası geçerli).*
